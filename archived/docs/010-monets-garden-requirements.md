# 010 — Monet's Garden: Project Requirements

> 컨셉·근거·시장은 `009-monets-garden-concept.md`. 이 문서는 **요구사항만** 적는다 (무엇을 만들어야 하는가).
> 표기: **MUST**(필수) / **SHOULD**(권장) / **WON'T**(이번 범위 제외). `❓`=미정.

## NS — North Star 제약 (모든 기능이 통과해야 함)
- **NS-1 (MUST)** 모든 기능은 *재미있고 · 인스턴트하고 · 판타스티컬*해야 한다. 셋 중 하나라도 깨면 채택 불가.
- **NS-2 (MUST)** 시전→첫 결과(외형 생성)는 체감 **즉각**(목표 수 초 이내). "Instant" 위반 = 불가.
- **NS-3 (MUST)** OVER-GAMIFY 금지(승패 게임화로 코지 톤 파괴 금지) **그리고** UNDER-DESIGN 금지(죽은 꾸미기로 얕아짐 금지).

## EXP — 코어 경험 (데일리 루프)
- **EXP-1 (MUST)** 데일리 루프: *시전(조합 발견) → 펑(탄생) → 정원에 자리·반응(생태) → 코지 배열*.
- **EXP-2 (MUST)** 시전/소환 순간에 **통쾌한 타격감**(화면 흔들림·터짐·사운드·무게감)을 준다. 코지 앰비언트와 충돌하지 않게 *순간 임팩트 + 잔향은 부드럽게*.
- **EXP-3 (MUST)** 두 시간축: **즉각 발아**(instant 환희) + **느린 개화/성장**(매일 돌아올 이유)을 한 메커닉이 동시에 제공.
- **EXP-4 (MUST)** 데일리 복귀 훅(새 마법진 · 변한 정원 · Monet이 해놓은 것)을 제공하되 **죄책감 금지**(안 와도 벌 없음, 오면 보상).
- **EXP-5 (SHOULD)** 코지 꾸미기는 *표현/안식 레이어*로 제공(본체 아님 — 깊이는 발견·생태에서).
- **EXP-6 (MUST)** **연쇄반응(cascade/chain)** — 한 행동이 연쇄를 촉발(캔디크러시 캐스케이드 · 앵그리버드 물리 연쇄 · Worms 폭발). 타격감(EXP-2)과 생태(ECO-2)의 핵심 동력: 시전/배치가 정원에서 줄줄이 터지는 통쾌함.

## LIFE — 생명/캐릭터
- **LIFE-1 (MUST)** "생명이 되다"는 **상태를 가진 반응형 개체**로 구현(타이머+CSS 흔들기 금지). 자라고·반응하고·기억하는 동작이 실제로 있어야 함.
- **LIFE-2 (MUST)** **소수의 깊게 authored 살아있는 존재**(니어 원칙). 무한 generative 종 양산 금지.
- **LIFE-3 (MUST)** **외형 = generative(정지 스프라이트), 모션 = 손으로 만든 공용 템플릿**(성장/흔들림/개화). 종마다 새 애니 클립 온디맨드 생성 금지.
- **LIFE-4 (MUST)** Monet은 **정서 화자** — 발견·개화를 *관계·감정 맥락*으로 반응(단순 사실 회상 금지). 애착 대상 = Monet, 정원 = 무대.
- **LIFE-5 (SHOULD)** 회화풍 아트 톤·알파 클린·무플리커 일관성을 모든 생성 외형에 보장(검수/필터).

## DISC — 발견 / 진행
- **DISC-1 (MUST)** 발견 메커닉: 룬/재료를 **조합**해 결과를 찾는다. *무엇이 나오는가 = 손-디자인 결정론 트리*(찾는 맛), *어떻게 생겼나 = generative*(나만의 것).
- **DISC-2 (MUST)** 발견이 **전진의 힘**: 새 조합이 새 룬·영역·정원 단계를 해금(진행 척추).
- **DISC-3 (SHOULD)** 하이브리드 보상: *모든 조합은 뭔가를 피움(dead-end 없음)* + *특정 조합은 특별 발견(희귀/열쇠)*.
- **DISC-4 (SHOULD)** 재귀 재료: 키운 식물이 다음 조합의 재료가 됨.
- **DISC-5 (MUST)** **슬롯머신 + 스킬 배합.** 입력=스킬(조합 발견), 출력=서프라이즈(generative 연쇄). 스킬이 변동보상의 *확률을 기울인다*(좋은 조합 = 더 희귀·더 큰 연쇄). 순수 슬롯도(="찾았다" 쾌감 죽음) 순수 스킬도(=스트레스) 아님. *이기면 내 실력, 운은 양념.* (research: 우연 스릴은 여전히 주요 결제 동인 — 슬롯 요소 유지.)

## ECO — 정원 / 생태
- **ECO-1 (MUST)** **아이소메트릭 정원**에 개체를 배치·관리. (`Stage.tsx`/`StackedVideo` 위에.)
- **ECO-2 (MUST)** 배치 = **생태 설계**(장식 아님): 개체 간 상호작용·유인·창발 이벤트로 *어디 두느냐가 의미를 가짐*.
- **ECO-3 (SHOULD)** 정원은 *살아서 surprise*하는 시스템(틱/상태/번식 등) — 정적 디오라마 금지.
- **ECO-❓** 생태의 결: *조용히 아름다움* vs *바글바글 생명이 깃듦* — 미정.

## SOC — 소셜
- **SOC-1 (MUST)** 친구 정원 **방문/구경**. (혼자 플레이 = 2주컷 방지.)
- **SOC-2 (SHOULD)** 신종/식물 **공유·선물**. 결과물의 *아름다움*을 외부 공유(획득 채널).

## MEM — 컴패니언 / 기억
- **MEM-1 (MUST)** Monet이 유저 취향·정원 이력을 **기억**하고 그것을 *관계*로 환원(LIFE-4와 결합).
- **MEM-2 (SHOULD)** 의존성/과몰입/안전 가드레일(컴패니언 역풍 방지).
- **MEM-❓** 메모리 인프라 build vs buy(Honcho 등) — deep-research 후 결정.

## MON — 수익 (상세는 deep-research 후)
- **MON-1 (MUST)** **마나 = 생성비에 페깅.** 무료 회복 마나 = *캐시/사전생성 풀(추론비 0)*, *진짜 generative 신종 소환* = 유료(씨앗/구독). 비용행위=결제행위 1:1.
- **MON-2 (MUST)** 비-착취적 코지 톤 유지(공격적 FOMO 금지) — NS와 충돌 금지.
- **MON-3 (SHOULD)** 모델 후보: 씨앗팩(가챠) · 햇빛/물(에너지) · 정원/온실 확장 · 마녀 구독 · 정원 스냅샷 소장.
- **MON-❓** 야심 크기(벤처 / IP·브랜드 / 인디) 및 구체 가격·전환 — deep-research 후.

## TECH — 기술
- **TECH-1 (MUST)** 기존 자산 위에: React + WebGL 스프라이트 스테이지(`v1/src/Stage.tsx`, `StackedVideo.tsx`), Cloudflare Workers + Hono(`v1/api`), R2.
- **TECH-2 (MUST)** 온디맨드 **정지이미지 생성**(SDXL류, 수 초). 온디맨드 비디오/애니 생성 의존 금지(LIFE-3).
- **TECH-3 (MUST)** 모션 템플릿(셰이더 sway/bloom 또는 소수 알파 모션 레이어)에 생성 외형을 끼워 재생.
- **TECH-4 (SHOULD)** 생성 외형 캐싱/사전생성 파이프라인(비용·latency·일관성 통제; docs/008 stacked-alpha 파이프라인 재활용).

## WON'T (이번 범위 제외 — 포커스용)
- **WON'T-1** 종마다 온디맨드 *비디오/애니메이션* 생성.
- **WON'T-2** 무한 generative 종 양산(소수 깊은 캐릭으로 대체).
- **WON'T-3** "게임"으로 포지셔닝/마케팅(게임 메커닉은 engagement 엔진으로만).

- 야심 크기 — `❓` (MON-❓).
- 생태의 결 — `❓` (ECO-❓).
- 메모리 build vs buy — `❓` (MEM-❓).
